# BUG-006: NestJS 스캐너 — `@Get([...])` 배열 경로 데코레이터 미지원 (대규모 누락)

- **상태**: Fixed (Phase135)
- **등급**: 정확도 결함 (false negative — 배열 경로 전부 소실)
- **현재 영향**: 실재 — NocoDB에서 ~330 라우트 누락
- **발견**: SaaS 추출 전수조사(NocoDB) 중 (2026-05-30)
- **파일**: `internal/scanner/nestjs/parse_decorator_call.go`

## 증상

`@Get(['/api/v1/x', '/api/v2/x'])`처럼 **경로를 배열로** 여러 개 지정하면
배열 안의 경로 문자열이 전부 소실되고 다중 endpoint로 확장되지도 않는다.

NocoDB 실측:
- controllers `*.controller.ts` HTTP 데코레이터 258개 중 **배열 경로 ~200개**.
  배열 항목까지 합산한 실제 라우트 ≈ **424개**.
- codist 추출(경로 올바를 때) **90개** → 약 **330개(78%) 누락**.
- 실증(`/tmp/repro/src/test.controller.ts`): `@Get(['/api/v1/arr','/api/v2/arr'])`
  → 출력 1개 `{"method":"GET","path":"/","handler":"arr"}`. 경로 2개 모두 사라짐.

## 근본 원인 (코드)

`parse_decorator_call.go` — `parseDecoratorCall()`의 `switch arg.Type()`에
`string/template_string/number/identifier/member_expression/object`만 있고
**`array` 케이스가 없다.** 배열 인자(노드 타입 `array`)는 무시되어 `d.args`가 비고
`d.arg=""` → `extract_one_method.go`의 `ep.path=d.arg`가 빈 문자열 → prefix만 남거나
`/`로 collapse.

## 권장 수정

`parse_decorator_call.go`에 `case "array":` 추가 — 배열 내부 string 자식들을
`d.args`로 수집하고, 호출부(`extract_one_method.go`)에서 path가 여러 개면
**각 경로마다 endpoint를 복제 생성**.
