# BUG-014: Django 스캐너 — 패키지형 모듈의 `include()` prefix 미합성

- **상태**: Fixed (Phase143)
- **등급**: 정확도 결함 (경로가 실제 URL과 다름 — prefix 누락)
- **현재 영향**: 실재 — Plane의 app/space/api 모듈 라우트에서 prefix 누락
- **발견**: SaaS 추출 전수조사(Plane) 중 (2026-05-30)
- **파일**: `internal/scanner/django/` (include 재귀 / prefix 합성부)

## 증상

`path('api/', include('plane.app.urls'))` 같은 `include()` prefix가 **패키지형 모듈**에는
합성되지 않아, 라우트 경로가 실제 URL보다 prefix가 빠진 형태로 방출된다.

Plane 실측:
- 단일 파일 모듈(`auth/`, `license`→`api/instances/`)은 prefix 유지.
- 패키지형 모듈(`app`→`api/`, `space`→`api/public/`, `api`→`api/v1/`)은 prefix **누락**:
  `/api/workspaces/...` 가 `/workspaces/...` 로 출력.
- 경로 recall(99.4%)·정확도엔 영향이 작으나 **경로 문자열이 실제와 불일치**.

## 근본 원인 (코드 — 검수 정정 2026-05-31)

> **정정**: prefix 합성(`expandURLEntry`→`combinePath`)은 패키지/단일파일을 구분하지 않고
> **균일**하다. "prefix 누적 전파 실패"는 메커니즘이 아니라 증상이다.

진짜 메커니즘:
- 패키지 `__init__.py`가 서브모듈 urlpatterns를 **star-import로 집계**
  (`from .workspace import urlpatterns as workspace_urls; urlpatterns=[*workspace_urls, …]`).
- `collectURLsFromFile`은 `path()` 호출만 수집하므로 `__init__.py`에는 entry 0건 →
  `byModule["plane.app.urls"]`가 비어 `include('plane.app.urls')`가 **빈 모듈로 해소**.
- 반면 서브모듈(`plane.app.urls.workspace` 등)은 아무도 include하지 않아 `findRootURLModules`에서
  **루트로 오인** → 빈 prefix로 전개 → `/workspaces/...`(=`api/` 누락). (실측: workspaces 206건 누락, `auth/` 36건 유지.)

## 권장 수정

- 패키지 include 해소 시 `__init__.py`의 star-import 집계(`from .X import urlpatterns`)를 따라
  서브모듈 entry를 패키지 모듈 키로 모으거나, `include('pkg')`가 `pkg.*` 서브모듈을 함께 전개.
  → 기존 `combinePath` prefix 합성이 그대로 적용됨(prefix 로직 변경 불요).
- 집계된 서브모듈이 `findRootURLModules`에서 루트로 오인되지 않게 include 대상으로 표시.
