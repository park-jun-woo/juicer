# BUG-007: NestJS 스캐너 — 스캔 루트에 `src/` 강제 + silent 0개 실패

- **상태**: Fixed (Phase136)
- **등급**: 사용성/정확도 결함 (silent failure — 경고 없이 endpoint 0개)
- **현재 영향**: 실재 — NocoDB를 `.../src` 경로로 주면 0개(100% 누락), 에러·경고 없음
- **발견**: SaaS 추출 전수조사(NocoDB) 중 (2026-05-30)
- **파일**: `internal/scanner/nestjs/find_controllers.go:13`, `internal/scanner/nestjs/scan.go:24-26`

## 증상

NestJS 스캐너가 **항상 `root/src/`만 순회**한다. 스캔 루트로 이미 `src`로 끝나는
경로(예: `packages/nocodb/src`)를 주면 `packages/nocodb/src/src`(존재하지 않음)를
찾아 빈 목록을 반환하고, 에러·경고 없이 **endpoint 0개**로 조기 종료한다.

NocoDB 실증:
- `scan packages/nocodb` (하위 `src` 있음) → **90개**
- `scan packages/nocodb/src` → **0개** (silent, 로그도 빈 상태)

## 근본 원인 (코드)

`find_controllers.go:13` `findTSFiles()`:
```go
srcDir := filepath.Join(root, "src")          // root 아래 "src" 강제
info, err := os.Stat(srcDir)
if err != nil || !info.IsDir() { return nil, nil }  // 없으면 0개
```
`scan.go:24-26`에서 빈 결과로 조기 종료. 경고 출력 없음 → silent failure.

## 권장 수정

1. `findTSFiles`: `root/src`가 없으면 `root` 자체를 순회(fallback), 또는 `*.controller.ts`가
   직접 보이면 그 디렉터리를 대상으로.
2. `scan.go`: TS 파일/컨트롤러 0개일 때 **경고 출력**해 silent failure 제거.

## 비고

- 다른 프레임워크 스캐너도 유사한 디렉터리 가정이 있는지 점검 권장.
