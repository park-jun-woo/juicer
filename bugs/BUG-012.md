# BUG-012: Django 스캐너 — `as_view({method:action})` 메서드 dict 무시

- **상태**: Fixed (Phase141)
- **등급**: 정확도 결함 (메서드 정보 붕괴 — 쓰기 메서드 대량 누락)
- **현재 영향**: 실재 — Plane에서 method recall 42.9%, POST/PUT/PATCH/DELETE 대량 누락
- **발견**: SaaS 추출 전수조사(Plane) 중 (2026-05-30)
- **파일**: `internal/scanner/django/url_entry.go:6-11`, `internal/scanner/django/resolve_call_arg.go:27-28`, `internal/scanner/django/build_url_mapped_viewset_endpoints.go:11`

## 증상

DRF의 URL-매핑 ViewSet `path('x/', XViewSet.as_view({"get":"list","post":"create","delete":"destroy"}))`
에서 **`as_view()`에 명시된 메서드 dict를 버린다.** URL에 정답(method)이 박혀 있는데도
무시하고 부모 클래스에서 메서드를 역산하려다, 부모 인식 실패(BUG-013) 시 단일 GET으로
추락한다.

Plane 실측: 누락 334건 중 **332건이 "경로는 찾았으나 메서드만 누락"**
(POST 148/DELETE 97/PATCH 74/PUT 13). 잘못된 GET 부여 79건.

## 근본 원인 (코드)

- `url_entry.go:6-11` — `urlEntry` 구조체에 **메서드-액션 매핑 필드 자체가 없음**.
- `resolve_call_arg.go:27-28` — `.as_view` 인지 후 view 이름만 추출, `{...}` dict 미파싱.
- `build_url_mapped_viewset_endpoints.go:11` — URL의 dict 대신 `resolveViewSetMethods(vs.parents)`
  로 부모에서 역산. URL에 박힌 정답을 무시.

## 권장 수정

1. `urlEntry`에 메서드-액션 매핑 필드 추가.
2. `resolveCallArg`에서 `as_view({...})` 의 dict 키(http method)를 파싱·전파.
3. `buildUrlMappedViewSetEndpoints`에서 부모 역산 대신 **dict의 method를 직접 사용**.

## 비고 (검수 2026-05-31)

- **BUG-013 의존**: Plane의 dict형 ViewSet은 대부분 커스텀 `BaseViewSet` 경유라
  `findViewSet` 인식 실패(BUG-013) 시 `buildUrlMappedViewSetEndpoints`에 **도달조차 못 한다.**
  따라서 "method recall 즉시 회복"은 BUG-013(상속체인 전이 해소)이 **선행돼야** 성립.
  직접 DRF 베이스를 상속한 dict형만 본 건 단독으로 회복된다. (구현 순서: BUG-013 → BUG-012)
