# BUG-009: 마운트 prefix 미합성 + `(method,path)` 단일 dedup으로 라우트 소실

- **상태**: Partially Fixed (Phase138, A만)
  - (A 완료) dedup 키에 출처 파일(File) 차원 추가 → 마운트 prefix가 합성되지 않아 서로 다른
    라우터가 같은 bare path로 방출돼도 출처가 다르면 모두 보존. Ghost content 16건 회복
    (content/routes.js 4→20, 전체 271→288). **라우트 소실 문제 해소.**
  - (B 미완) Ghost 비표준 마운트(`apiApp.lazyUse('/content/', require('./content/app'))` +
    팩토리 export 라우터 `module.exports = function apiRoutes(){…}` + 중첩 app `apiApp.use(routes())`)
    해소를 통한 **prefix 합성은 미구현**. 회복된 content 라우트는 여전히 bare path
    (`/posts`, prefix `/content/` 미부착). 사유: lazyUse 커스텀 마운트 메서드 인식 + 인라인
    require 마운트 인자 해소 + 팩토리-함수 export → (file,var) 매핑 + app↔router 2-hop 중첩
    마운트 해소가 필요한 상당 규모 작업으로, 본 트랙 scope·난이도를 초과(별도 Phase 권장).
- **등급**: 정확도 결함 (false negative — 서로 다른 prefix의 동일 path 충돌)
- **현재 영향**: 실재 — Ghost content API 16 라우트 소실
- **발견**: SaaS 추출 전수조사(Ghost) 중 (2026-05-30)
- **파일**: Express 마운트 해소부(`extract_use_mount.go`/`parse_use_mount_args.go`) + `internal/scanner/deduplicate_endpoints.go:11`

## 증상

서로 다른 마운트 prefix 아래의 동일 path 라우트가 prefix 미합성 + dedup으로
하나로 합쳐져 소실된다.

Ghost 실측:
- admin 라우터(`/ghost/api/admin`)와 content 라우터(`/ghost/api/content`)의 마운트
  prefix가 **해소되지 않아** 둘 다 bare `/posts`로 방출.
- `DeduplicateEndpoints`가 같은 `(GET, /posts)` 키로 보고 **content 쪽 16건을 폐기**.
- 생존한 4건은 admin에 동일 path가 없어 충돌하지 않은 것뿐 → content recall 4/20.

## 근본 원인 (코드 — 검수 정정 2026-05-31)

> **정정**: prefix 합성 자체는 **이미 동작한다**. `resolve_use_prefix.go`→`build_mount_graph.go`
> →`resolve_router_prefixes.go`/`propagate_prefixes.go`→`build_endpoints_from_file.go:13`이
> 표준 `app.use('/api/content', router)`(router=식별자, 타 파일 `express.Router()` 생성·
> `module.exports`)에서 정상 합성한다(실증: `/api/admin/posts`·`/api/content/posts` 둘 다 보존,
> dedup 충돌 없음). 따라서 "dedup이 16건 소실의 1차 원인"은 정확하지 않다.

Ghost에서 prefix가 합성되지 않는 **진짜 1차 원인은 비표준 마운트 미해소**다:
1. 커스텀 마운트 메서드 `apiApp.lazyUse('/content/', …)`(`web/api/app.js:26-27`) — `extract_use_mount.go`가 `use`만 인식.
2. 마운트 인자가 식별자가 아닌 인라인 `require('./endpoints/content/app')` — `parse_use_mount_args.go`는 마지막 인자가 `identifier`일 때만 처리.
3. 라우터 모듈이 인스턴스가 아닌 **팩토리 함수** export(`module.exports = function apiRoutes(){ … return router }`) — (file,var) 매핑 불가.

prefix가 안 붙으니 둘 다 bare `/posts`가 되고, 그때 비로소 `deduplicate_endpoints.go:11`의
`key{ep.Method, ep.Path}` 충돌로 content 16건이 폐기된다(2차 증상).

## 권장 수정 (택1, scope 주의)

- **(A 보수적)** dedup 안전망: `deduplicate_endpoints.go:11` 키에 출처(파일)를 더하거나
  동일 (method,path)라도 출처가 다르면 보존 → 소실은 막으나 path는 bare 상태.
- **(B 근본)** Ghost류 비표준 마운트 해소: 커스텀 mount 메서드(`lazyUse`) 인식 + 인라인
  `require()` 마운트 인자 해소 + 팩토리-함수 export 라우터 (file,var) 매핑. **상당 규모** —
  단독 작업으로 분리 권장.

## 비고

- 표준 `app.use(prefix, identifier)` 합성 경로는 이미 동작하므로 불변.
- Django의 패키지형 include prefix 결함은 별개 원인(BUG-014, `__init__.py` 집계).
